React and Next.js support must be explicitly enabled — they are not auto-detected.

## React

<Steps>
  <Step title="Enable React in your config">
    ```js eslint.config.mjs
    import antfu from '@antfu/eslint-config'

    export default antfu({
      react: true,
    })
    ```
  </Step>
  <Step title="Install required dependencies">
    <CodeGroup>
      ```bash npm
      npm i -D @eslint-react/eslint-plugin eslint-plugin-react-refresh
      ```
      ```bash pnpm
      pnpm i -D @eslint-react/eslint-plugin eslint-plugin-react-refresh
      ```
      ```bash yarn
      yarn add -D @eslint-react/eslint-plugin eslint-plugin-react-refresh
      ```
    </CodeGroup>

    Running `npx eslint` will also prompt you to install these automatically.
  </Step>
</Steps>

The React integration enables rules from `@eslint-react/eslint-plugin` under the renamed prefixes `react/`, `react-dom/`, `react-naming-convention/`, `react-rsc/`, and `react-web-api/`. It also includes `eslint-plugin-react-refresh` under the `react-refresh/` prefix for fast-refresh compatibility checks.

## JSX accessibility

JSX accessibility rules are provided via `eslint-plugin-jsx-a11y`. Enable them through the `jsx` option:

```js eslint.config.mjs
import antfu from '@antfu/eslint-config'

export default antfu({
  jsx: {
    a11y: true,
  },
})
```

Install the required dependency:

<CodeGroup>
  ```bash npm
  npm i -D eslint-plugin-jsx-a11y
  ```
  ```bash pnpm
  pnpm i -D eslint-plugin-jsx-a11y
  ```
  ```bash yarn
  yarn add -D eslint-plugin-jsx-a11y
  ```
</CodeGroup>

## Rule overrides

Use the `overrides` key inside the `react` or `nextjs` option to customize specific rules:

```js eslint.config.mjs
import antfu from '@antfu/eslint-config'

export default antfu({
  react: {
    overrides: {
      'react/prefer-namespace-import': 'warn',
      'react-refresh/only-export-components': 'off',
    },
  },
  nextjs: {
    overrides: {
      'next/no-html-link-for-pages': 'warn',
    },
  },
})
```

<Note>
  Plugin prefixes are renamed from their npm package names. Use `react/` instead of `@eslint-react/`, and `next/` instead of `@next/next`. See the [plugin renaming reference](/reference/plugin-renaming) for the full mapping.
</Note>

## UnoCSS

[UnoCSS](https://unocss.dev/) linting support must be explicitly enabled.

<Steps>
  <Step title="Enable UnoCSS in your config">
    ```js eslint.config.mjs
    import antfu from '@antfu/eslint-config'

    export default antfu({
      unocss: true,
    })
    ```

    For advanced configuration:

    ```js eslint.config.mjs
    import antfu from '@antfu/eslint-config'

    export default antfu({
      unocss: {
        // Enable attributify ordering rules (default: true)
        attributify: true,
        // Enable strict mode — error on blocklisted classes (default: false)
        strict: false,
      },
    })
    ```
  </Step>
  <Step title="Install required dependencies">
    <CodeGroup>
      ```bash npm
      npm i -D @unocss/eslint-plugin
      ```
      ```bash pnpm
      pnpm i -D @unocss/eslint-plugin
      ```
      ```bash yarn
      yarn add -D @unocss/eslint-plugin
      ```
    </CodeGroup>
  </Step>
</Steps>

The UnoCSS integration enables rules under the `unocss/` prefix:

- `unocss/order` — warn on incorrect class order
- `unocss/order-attributify` — warn on incorrect attributify class order (when `attributify: true`)
- `unocss/blocklist` — error on blocklisted classes (when `strict: true`)
